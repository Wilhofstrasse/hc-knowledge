2 — Was du gleich tun kannst

  1. Schritt 1: Template extrahieren. Lade ein fertig produziertes Kunden-PowerPoint (.pptx) hoch. In ~3 Sekunden bekommst du drei Dateien zurück: template.pptx (deine "Skelett-Datei", alle Texte entfernt aber Layout intakt), template.json (maschinenlesbare Struktur, 1-2 MB), und template.md (kurzer Lese-Digest pro Folie).
  2. Schritt 2: Für neue Firma regenerieren. Gib einen neuen Firmennamen ein (optional Branche, Größe, Geographie, Stichworte). Klick → Gemini schreibt jeden Textbaustein auf die neue Firma um. ~30 Sekunden später hast du das neue Deck.
  3. Was zu erwarten ist: Bei dem 5-Folien Test-Deck (H_C_Master) sind alle 30 Textshapes gefüllt worden, 0 Fehler. Bei dem 10-Folien Dummy-Deck waren es 91 Shapes, ebenfalls 0 Fehler.
Sanity check vor dem ersten Kunden-Deck: Erst mit dem Test-Deck testen (H_C_Master oder Dummy), schauen wie das Ergebnis aussieht, dann erst echtes Kundenmaterial. Ist halt v0.1, kein v1.0.

Probleme die in deinem Output sichtbar werden

#Was passiertWann es zuschlägtSchwere
1 Mehrere Aufzählungspunkte in EINEM Textfeld werden zu einem einzigen Satz zusammengezogen. Die Bullets verschwinden. Nur wenn ein Textfeld mehr als einen Absatz enthält. Das Test-Deck hat das nicht — jedes Bullet ist dort sein eigenes Textfeld. Echte Kunden-Decks sind oft anders gebaut. P1
2 Text der innerhalb einer "Gruppe" liegt (z.B. Icon + Label gruppiert) wird gelöscht aber NICHT neu geschrieben. Bleibt im Output leer. Nur wenn das Quell-Deck gruppierte Shapes mit Text hat. Test-Decks sind frei davon. P2
3 Wenn eine LLM-Anfrage mittendrin scheitert (Netz-Fehler / Rate-Limit), hat das fertige Deck einzelne leere Shapes. Selten. Wir zeigen jetzt eine gelbe Warnung im UI, sobald das passiert. P2 — gefixt im UI
4 Schriftart und -größe können auf Default zurückfallen (Calibri 18pt) wenn das Original-Shape leere Runs hatte. Edge-Case bei manchen leer aussehenden Quell-Shapes. Unwahrscheinlich beim Test. P2

#Was passiertWirkung

Externe Cross-Modell Calibration (5 Reviewer-AIs, $0.20 Spend)

Fünf top-tier Coding-Models (DeepSeek v3.2, Gemini 3.1 Pro, GPT-5.4, Claude Sonnet 4.6, Claude Opus 4.7) haben den gleichen Code unabhängig durchgesehen. Wenn mehrere AIs dasselbe Problem flaggen, ist es real (nicht nur die Macke einer einzelnen Meinung).

#Konsens-FindingAIs einigSchwere
A Netzwerk-Calls haben kein Retry. Sobald OpenRouter mal stockt (HTTP 429, 503), gibt der Spike sofort auf — diese Folie bleibt leer. Fix: One Helper-Function mit 3 Retries auf transiente Fehler. 5/5 P1
B Shape-Matcher ist kaputt bei Gruppen. Wenn eine Folie eine "Gruppe" (Cluster aus mehreren Shapes) enthält, verschiebt sich JEDER folgende Shape-Match um eine Position. Das Deck wird mit den FALSCHEN Texten in den FALSCHEN Boxen gefüllt — silent. Fix: nach shape_guid matchen statt nach Position, oder beidseitig rekursiv durch Gruppen laufen. 4/5 P1
C Die LLM-Antwort wird ohne Validierung in das Deck geschrieben. Wenn das Modell Mal kein gültiges JSON oder die falsche Struktur zurückgibt, crashed entweder die Folie oder bleibt leer — kein Banner, kein Retry. Fix: kleine Schema-Prüfung vor dem Apply. 4/5 P1
Wichtigste Take-Away aus der Calibration: der Multi-Paragraph + per-Run-Formatting Bug (4/5 Models, P2) verlangt nach Ansicht der Reviewer keinen Patch — sondern einen Port der bereits funktionierenden Fill-Logik aus dem alten corporate-ppt-tool-v2-Repo. Dort ist das Problem schon gelöst. Spart einen halben Tag.
