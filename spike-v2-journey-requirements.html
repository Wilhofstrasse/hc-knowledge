Diese Seite spiegelt die 11-Frame Consultant-Journey wider und zeigt unter jedem Frame, was das Add-in stillschweigend prüft, was es dem Consultant sichtbar macht, und wann es eine Aktion ablehnt oder blockiert. Zweck: ein einziges Sign-off-Surface für den spike-v2 trust contract — lesbar in 30 Sekunden auf dem iPhone.

The wireframe is the visual sign-off layer for the partial-update primitive. Frames 7, 8 and 10 carry the load — mark, fill, and durability verify. Everything else is journey context.

Silent — invariants the consultant never sees unless they fail.
Surfaces — visible UI state: buttons, status pills, copy.
Refuses — eligibility rejections, kill criteria, durability failures.
Pending durability — written but not yet round-tripped.
Durable — save+reopen verified, byte-diff clean.

Frame 07

Consultant prüft einzelne Form — Mark step

Mark · load-bearing

Silent — checks & reads

  • checkMarkEligibility(shape) runs 4 gates:
  • (a) shape type not in {table, chart, smartArt, ole, graphicFrame, group, picture, image}.
  • (b) textFrame.hasText is true.
  • (c) No \v\n\r\f in textRange.text — exactly one paragraph.
  • (d) All 12 ShapeFont attributes non-null — homogeneous-format runs.
  • On pass: format_signature_hash computed and persisted alongside shape_guid.

Surfaces — visible UI

  • Selected shape highlighted on the slide.
  • Mark whole-shape target button enabled.
  • Brief prompt input, max 500 characters.
  • Success: Marked target_id=4f2a…
    Copy: „Marked. Use ‘Fill from spreadsheet’ next.“

Refuses — blocked actions

  • Spike-v2 does not support shape type table; mid-band only.Gate (a) — eligibility
  • Shape has no text content; nothing to fill.Gate (b) — textFrame.hasText
  • Spike-v2 requires exactly 1 paragraph; shape has N.Gate (c) — single-paragraph
  • Shape has mixed-format text runs (mixed: name, underline, strikethrough); spike-v2 requires a single run.Gate (d) — homogeneous format
  • Brief exceeds 500 chars (got N).Brief length cap

Frame 08

Consultant startet die KI-Befüllung — Fill step

Fill · load-bearing

Silent — checks & reads

  • Re-fetch UpdateTarget row by shape_guid.
  • Re-validate format_signature_hash at fill time — guards against edits between mark and fill.
  • Compute prev_text_hash = SHA-256 of textRange.text.
  • RAG retrieval returns Evidence[] — top-5 with embedding_score + locator.
  • Confidence gate: delta_to_runner_up ≥ 0.15 → autofill, else picker.

Surfaces — visible UI

  • Fill from spreadsheet (selected target) button click.
  • Floor-band: prompt for proposed_value.
  • Mid-band: Evidence picker modal — 5 candidates with source locators.
  • State on completion: Pending durability
    Copy: „Pending durability — save+reopen then verify.“

Refuses — blocked actions

  • Shape is not marked. Call markWholeShape first.No UpdateTarget row for this shape_guid
  • Shape is no longer eligible — was edited between mark and fill. Re-mark required.Eligibility re-check failed
  • Format drifted between mark and fill — baseline {abc12345}, current {xyz67890}. Re-mark required.format_signature_hash mismatch (Codex round-3 fix)
  • delta_to_runner_up < 0.15 — picker required (mid-band).Confidence gate

Frame 10

Consultant prüft und speichert — Durability verify

Durability · load-bearing

Silent — checks & reads

  • Cmd+S → save + close + reopen.
  • scripts/pptx-bytediff-v2.py before.pptx after.pptx --target-slide N --target-xpath './/a:t' --mutation-kind text-only exits 0.
  • finalizePatchAfterDurabilityCheck promotes the pending audit row to update_target_filled_durable.
  • In-deck claim_ledger row written with state: "source_backed".
  • state.json checkpoint advances to completed.

Surfaces — visible UI

  • Save confirmation; reopen confirmation.
  • Sidebar log: spike_v2: filled-durable target_id=…
  • State pill flips pending → Durable.
  • Audit panel shows the full provenance trail: target → claim → evidence locators.

Refuses — blocked actions

  • byte-diff exit 1: UNEXPECTED DRIFT — diff in non-whitelisted part.Rollback row written; consultant must investigate which part drifted
  • byte-diff exit 3: INVARIANT VIOLATION — whitelisted part diff exceeds mutation_kind scope.Rollback row written; spike escalates to architecture rethink per kill criteria
  • claim_id cross-reference resolution failed — audit walker reconciliation pending.Audit-trail integrity gate

Kill criteria roll-up

Eight trigger conditions, each with the agreed resolution path. Severity flags the partners who get pulled in: amber means iterate within the spike; red means stop and escalate to Filipe.

Trigger Resolution Severity
If the add-in cannot read the file you picked. We pivot to a small local helper that talks to the add-in over localhost. Acceptable. Acceptable iterate
If marking succeeds but PowerPoint silently strips our identity tag on save. We retry using an out-of-band tag-store keyed by content hash. Acceptable. Acceptable iterate
If the format signature drifts on every reopen (false positive re-mark spam). We tighten the signature to only the 4 attributes Office actually round-trips. Acceptable. Acceptable iterate
If the byte-diff says clean but the deck visibly differs on reopen. Stop. Escalate. The trust contract is broken; spike pauses for architecture rethink. BLOCKER
If the byte-diff says dirty but the deck is visually identical (false positive drift). We expand the whitelist by one path-class with explicit reviewer sign-off. Acceptable. Acceptable iterate
If the RAG retrieval cannot find a source-backed value for any cell. We fall back to the picker modal; consultant supplies the value with a manual locator. Acceptable. Acceptable iterate
If durability verification consistently fails on Mac PowerPoint but passes on Windows. Stop. Escalate. The spike cannot ship for the consultant population without round-trip parity. BLOCKER
If audit-trail walker reconciliation fails on a deck the partner is about to send. Stop. Escalate. Refuse the save; the consultant must re-fill the target manually. BLOCKER
