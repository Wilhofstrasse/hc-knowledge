Grounding LOCKED v2.1 v2.1.1 amendment pending

The proof layer for every textual statement in the deck. Every claim has a citation. Every citation gets verified.

Plain-English summary. When the AI writes a number or statement on a slide, where did it come from? The grounding module is the answer. It ingests source materials at deck open (PDFs, EDGAR filings, IR docs, client uploads), retrieves the relevant chunks for each shape, asks the AI to write text grounded in those chunks, and validates that the AI's output is actually backed by the cited source — not just "looks like a citation." Every claim that survives this gate gets a green / yellow / orange / red traffic-light in the consultant's add-in pane.

What it does

Comment on §What it does
  • Reads the consultant's source materials — at deck open, parses every uploaded PDF and external feed into searchable chunks with full provenance.
  • Retrieves chunks for every fill — when the consultant clicks a shape and asks the AI to fill it, the right source chunks get surfaced via search.
  • Validates the AI's output — the AI writes text and citations; the validator checks (a) citations point to real chunks, (b) the cited text is verbatim in the source, and (c) the chunk semantically supports the claim (not just substring-match).
  • Re-asks if validation fails — up to 3 retries; if all 3 fail, claim gets blocked (RED tier) and the consultant is told why.
  • Surfaces traffic-lights in the add-in — every shape gets a confidence-tier badge: green / yellow / orange / red. Click for paragraph-level citations like Claude.ai or ChatGPT.

The 4-tier architecture

Comment on §The 4-tier architecture

Tier 0 — Source ingestion (~3-5d build)

Comment on §Tier 0 — Source ingestion (~3-5d build)

Uses docling (MIT license, viable for SaaS resale; deliberately NOT marker-pdf which is GPL+commercial). Parses PDFs, DOCX, HTML into structured chunks at paragraph granularity. Each chunk gets a content hash, page reference, and full provenance. Encrypted at rest in tenant-scoped R2 with FTS5 full-text-search index.

Tier 1 — LLM call with grounded prompt (~3-5d per provider adapter)

Comment on §Tier 1 — LLM call with grounded prompt (~3-5d per provider adapter)

Filipe's #3 feedback reshapes this tier. Original spec made Anthropic's native Citations API the architectural spine — meaning the architecture was vendor-locked to Anthropic. Filipe flagged this as wrong: "it needs to be model agnostic."

Amended (pending v2.1.1 sign-off): the spine becomes the universal CitationEnvelope contract that every provider adapter must produce. Anthropic Citations API, Google groundingSupports, OpenAI Structured Outputs — all become adapter optimizations. When the active provider supports them, the adapter uses them; when not, the adapter falls fully through to the entailment validator. The architecture doesn't depend on any one provider.

Tier 2 — Validation + re-ask (~2d build)

Comment on §Tier 2 — Validation + re-ask (~2d build)

Wraps every LLM call with Pydantic schemas via the instructor library. Two checks run on each citation:

  • Substring match (cheap): is the cited_text actually verbatim in the chunk?
  • Semantic entailment (NEW per v2.1.1): a small NLI cross-encoder (e.g., cross-encoder/nli-deberta-v3-base) verifies the chunk semantically supports the claim, not just contains the words. Threshold 0.70 default; tenant-configurable.

If either fails, instructor re-asks the LLM with corrective feedback. Up to 3 retries. After 3 fails: claim → state: blocked, tier: RED, audit row written, consultant sees why.

Tier 3 — Add-in pane UX (~3-4d build)

Comment on §Tier 3 — Add-in pane UX (~3-4d build)

Always-visible sub-pane in the add-in shows: traffic-light per shape, source URL, paragraph excerpt, citation snapshot. 5 quick-buttons (regenerate, override, attest, refresh source, show full Cockpit). Full Grounding Cockpit modal (~900×600) for partner review pane with revision history, send-gate enforcement, source freshness alerts.

Tier 4 — Audit walker (~2d build, OPS phase)

Comment on §Tier 4 — Audit walker (~2d build, OPS phase)

Background process that periodically walks every deck checking for: orphan claims (text changed but no revision-log row), stale sources (citation URL returns 404 or content hash drift), broken citation pointers, expired licenses on Imagery. Surfaces RED in Cockpit; doesn't block fills, but flags drift.

Cost

Comment on §Cost

~$0.50-1.00 per deck for grounding alone (Anthropic adapter). ~$2.5-5K per tenant per year dominated by Anthropic API costs. Per-provider variance after v2.1.1: Gemini ~$0.50-1.20/deck (cited text counted as output tokens, unlike Anthropic); OpenAI ~$0.80-1.80/deck. Per-deck budget cap in cost-governance §G.

Critical caveats

Comment on §Critical caveats
  • Anthropic Citations API and Structured Outputs are mutually exclusive — confirmed at API level. Anthropic adapter outputs plain text + citations array; structured-shape parsing happens in Tier 2 instructor, not Anthropic JSON-schema-mode.
  • Gemini grounding is public-web only — Gemini's groundingSupports[] doesn't cover private corpora. For private chunks, Google adapter falls through to Tier 2 entailment validator. Architecturally clean; cost goes up.
  • EU-only tenants route via Anthropic Bedrock EU geo — never silent failover to US (per v2.0 §I).
  • F2 latency budget: ≤200ms p95 with session-trust ON (cached); ≤2.5s p95 classifier; ≤8s p95 full pipeline. Session-trust default ON for F2 edits.
