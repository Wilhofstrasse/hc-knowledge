Locked decisions

Atlas keeps one source of truth per repository. Every change to a project — whether you write it in plain English here or someone pushes code on GitHub — flows through the same record. There is no second place to look.

Each project carries a state — red when there is a question waiting for an answer, orange when an answer is ready for you to read, green when everything has been signed off. The colours are barely there; reading should dominate.

Building right now

The logbook now fails closed. If the database is unreachable when a write comes in, Atlas refuses the write and surfaces the failure to the caller, instead of retrying silently and losing the change. read answer ⌄

answer · ready

Codex moved the failure mode from open to closed in the logbook contract. The behaviour is now: if the database can't be reached, Atlas returns an error to whoever sent the write, and the write does not happen. Six sparring agents reviewed; one flagged a small race condition near the flush path — the fix is queued and visible on the run timeline.

Sign-off now requires Face ID by default on phones. The earlier behaviour — sign-off via tap with no second factor — is preserved on desktop, where typing the passphrase serves the same role.

Waiting on you

Logbook retention. Today, every write is kept forever. Storage cost is negligible at the current pace, but there is no policy on the books, and the GDPR review last quarter flagged it as something to decide before the next audit.

Read-only mirror for partners. Two clients have asked to see the logbook live. We can give them a read-only link without giving them edit rights, but the routing is not in place yet — that's the next visible piece of work.
