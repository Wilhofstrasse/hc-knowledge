1. Where we are right now

Comment on §1. Where we are right now

Three things changed this week and they layer on each other:

  1. The architecture is locked. After multi-agent review (3 specialist agents + 3 Codex iterate rounds → GREEN) the fill-engine architecture is no longer a draft. Five primitives (fill, stamp, detect, match, restructure) + four invariants (stable shape id, authorship attribution, audit trail, consent revocation) + a 2×3 user-flow matrix replace the misframed three-mode taxonomy from earlier in the week.

3. Architecture in one picture

Comment on §3. Architecture in one picture

The fill engine is built from five primitives. User-facing flows are compositions of these primitives. Four invariants run across all of them.

P1
fill — single shape gets new text from the AI
Built and Phase 0 PROVEN. Used by every flow.
P2
stamp — drop a template's geometry onto a slide
Not built. The dummy-deck workaround substitutes for it today.
P3
detect — classify shapes on a slide (user-typed vs AI-filled vs empty), propose roles
Not built. The consultant's daily-reality primitive.
P4
match — given source materials + library, propose a storyline + template per section
Not built. Jumpstart's hard problem.
P5
restructure — delete / move / merge / reorder slides at deck level
Not built. Re-thread requires this.

Four invariants (run across every primitive)

I1
Stable shape identity — UUID stamped to every shape on first read, first-write-wins on collision
Implemented in addin/src/invariants.ts · ensureShapeGuid()
I2
Authorship attribution — every shape carries hc_filled_by ∈ {user, ai, null}, default-to-user on ambiguity
Implemented · ensureHcFilledBy() / markAiFilled() / markUserAuthored()
I3
Audit trail — append-only log in the deck, 4 KB inline prompt + ref for larger (offline-readable)
Implemented · appendAuditEntry() / readAuditBody()
I4
Consent revocation — purge primitive strips every hc_* tag + urn:hc-* XML part
Implemented · purgeAllHcArtifacts() · Phase A pattern-match version

Where the AI call goes (today)

Sidebar in PowerPoint ──> Local proxy ──> OpenRouter ──> Gemini 3 Flash (Filipe's Mac) (cloud router) (Google) │ Sidebar applies text <── Local proxy <── OpenRouter <──────────┘ parses JSON writes audit row to deck Production swap (later): Local proxy → H&C backend Worker (Option C). Same surface. Centralised key. Per-user auth. Audit log + cost ceilings.
