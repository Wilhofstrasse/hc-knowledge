5. Decisions I made on your behalf

Each is reversible if you want.

#DecisionWhyHow to reverse
1 Use OpenRouter directly, not the router-mcp Worker The credentials path the Worker expects didn't exist on this Mac. The default OpenRouter key was rotated. openrouter-frankenstein-key was the only live one. Re-route through router-mcp later if you want centralized audit + budget tracking.
2 Use Gemini 3 Flash (Google), swappable Started with Claude Sonnet 4.6 overnight, swapped to Gemini Flash on your morning request — ~3x cheaper, ~30-40% faster, similar output quality. Production model choice belongs to the H&C backend when it ships. One-line model swap at addin/server/fill-proxy.ts line 74.
3 "Fill slide" (whole-slide) is the primary path Phase 0 last week proved this beats "Fill shape" on coherence. Use the "Fill shape" button if you want single-shape demos.
4 Match slide shapes to template by reading order Quick to ship; works when slide and template have the same shape count. Production fix would tag each shape with a stable id (logged as Q-LAYOUT-01).
5 Worked on a feature branch, not main So you can review every commit before merging. Merge feature/test-version-fill → main when satisfied.

6. Caveat — read this before the demo

Ordinal mismatch on the test deck The "Fill slide" button matches Office.js shapes to template-shape records by reading order (1st on slide → template's 1st, etc.). Your test deck doesn't match any template cleanly.
SlideText shapesClosest templateNotes
15thesis (6) — closefirst shape may not be the title — quirky placement
25thesis (6) — closesame
36thesis (6) ✓Best for demo
49nonespike-test slide, too many shapes
52cover (3) — closetoo few

Best-case demo: slide 3 + thesis template. Even so, the shape order in PowerPoint may not exactly match the template's intended order, so you might see "title-slot" content land in a "bullet-slot" shape. Output is still H&C-quality copy — just possibly in the wrong placement.

Truly clean output: add a fresh slide with 6 text boxes laid out top-to-bottom (one big title, four bullets, one footer). Then ordinal binding aligns perfectly.
