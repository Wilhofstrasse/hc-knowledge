The Logbook module — one module, replaces four

Comment on §The Logbook module — one module, replaces four

Per your #6: "prompt registry = logbook... we need a module to encapsulate all of this."

We had four separate concepts: (a) what's currently true on each shape, (b) every action ever taken, (c) history of edits per shape, (d) which prompt produced which fill. They all said the same thing: we record what happened in the deck. So now: ONE module — the Logbook.

  • One entry shape — actor (user/AI), action (fill/override/regen/attest), before/after text, prompt used, source citations, reason, timestamp.
  • Two physical homes — inside the .pptx file (travels with the deck) + in a project library (searchable across decks).
  • Append-only by construction — entries can be added, never edited. That's the trust story. No regulator framing.
  • Net build saving — ~9 days for the unified module vs ~25-30 days for the four separate ones.

Draft is in the holding folder waiting for your sign-off.
