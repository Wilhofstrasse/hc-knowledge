Architecture — plain-English reader

For Filipe, partners, clients, and anyone who needs the picture before the spec. Three modules, one foundation.

What this is. A multi-tenant PowerPoint add-in for consulting-grade decks. Consultants click a shape, the AI fills it with text grounded in real sources, every claim has a traffic-light, every action is logged for audit. Built for regulated industries — banking, insurance, asset management, pharma — where "show me where this number came from" is not optional.

Why this page exists. The full architecture has 10 invariants, 6 primitives, 6 user flows, and ~200 pages of locked specs across multiple files. That's right for engineers and Codex. It's wrong for the partner who needs the picture in 5 minutes. This page summarizes the architecture in three modules + one foundation, with links down for verification.

The three user-facing modules

⚖️ Trust apparatus LOCKED v2.1 v2.1.1 pending

The system that tracks what's true in your deck and who said it. Library (vetted templates), Claim Ledger (current truth per shape with traffic-light), External Audit (tamper-proof action log, 7-10y retention), Disclaimer Flag (gap-honesty fallback footer).

Architecturally one apparatus, four invariants. The audit committee's answer to "show me you didn't make this up."

Open: Q1 multi-vendor MVP shape, Q2 revision logbook storage, Q3 partner sign-off state machines, 7 Q-V20 defaults.

→ Read the Trust apparatus page

🖼️ Imagery LOCKED v2.1

All non-text visual content the AI fetches or proposes — cover photos, competitor/peer-firm logos for peer analysis, country flags + sector icons for PESTEL slides, peer-map firm icons. One module, broad scope.

6-tier picker for cover photos: user-upload → IR press kit → H&C curated library → licensed stock (Shutterstock + Getty) → AI-gen (illustrative only, NEVER fake real-firm building photos) → H&C placeholder.

Out of scope: employee/people photos (privacy + GDPR — manual upload only, not AI-fetched).

→ Read the Imagery page

🔍 Grounding LOCKED v2.1 v2.1.1 pending

The proof layer for every textual statement in the deck. Sources get ingested at deck open, retrieved via search, fed to the AI as grounded prompts, validated by structured-output schemas + semantic entailment, surfaced as traffic-lights + paragraph-level citations.

Filipe's model-agnostic feedback (#3) reshapes this: the spine is the universal citation envelope + entailment validator, NOT any one provider's API. Anthropic / Google / OpenAI become adapter optimizations.

Open: 15 Q-GR defaults, semantic entailment NLI eval set, multi-vendor cost attribution.

→ Read the Grounding page

🔐 Plumbing (foundation) LOCKED v2.1

The invisible layer that makes everything else regulated-grade. Per-tenant data isolation (AAD-bound AES-256-GCM encryption), Azure Key Vault for tenant master keys, schema-versioning for forward-compatible migrations, region routing for EU-only tenants.

You won't read this page — you'll never see it as a consultant. Auditors will read the L1 Atlas, not the L3 reader. Skipped here intentionally.

For the spec: see Tenant ID (Inv #5), Schema-versioned (Inv #10), Vendor region + data residency (Inv #9).

How they connect

Source materials → Grounding (ingestion + retrieval + adapters) → Claim Ledger (with verdict tier) → Add-in UI (consultant sees traffic-light) → External Audit (every action logged). Imagery flows in parallel for visual shapes. Plumbing wraps every payload at every step.
