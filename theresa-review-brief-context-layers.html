What the spec actually says, in 5 minutes

The mental model

Three "roles" each contribute context to every AI fill:

RoleWhoWhat they contribute
i — PowerPointThe base layerShape geometry, font, position, the deck's overall theme. This already exists today.
ii — AdminThe library curatorWhat this slide template is for, what guidance applies, what format constraints (e.g., "verbs in bold"). Brand-new in this spec.
iii — UserThe consultant filling the deck"For this specific deck, override the library — use Bain not Bloomberg as primary source." Brand-new in this spec.

Three "context levels" — deck / slide / shape — and two dimensions — content (what to say) and format (how to say it). That gives a 17-cell grid covering everything the AI could be told.

The big rules

  • No silent overrides. If a consultant overrides a library default, the spec defines exactly how that's recorded so the next person looking at the deck can see "this was changed and why".
  • Shape identity must survive copy-paste, slide duplication, and deck-to-deck transfers. The spec defines a unique stable identifier per shape that doesn't break when consultants do normal PowerPoint things.
  • Schema versions are negotiated up-front. When the add-in starts, it asks the AI service "what version do you understand?" and locks that for the session. No mid-session surprises.
  • Tombstones for deleted guidance. If a consultant says "ignore this admin guidance", the deck records that suppression in a way that survives even if the original admin entry later gets renamed or deleted.
  • Conflict surfacing. If the admin says "verb in bold" but the AI generates content with no verb, the consultant sees a clear modal: drop the format constraint, drop the content override, or cancel. Never silent.

What it explicitly does NOT do

  • It does NOT add the admin authoring UI yet (deferred to a later wave).
  • It does NOT add multilingual authoring yet (also deferred).
  • It does NOT change the trust contract from the safety-check spec — those guarantees stay intact.
